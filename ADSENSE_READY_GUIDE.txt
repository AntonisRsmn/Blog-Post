Google AdSense - Ready Guide (Enable later)

Your website is now prepared with placeholder ad slots on all frontend pages.
Ads will NOT run until you replace placeholder IDs in:
- frontend/js/adsense.js

--------------------------------------------------
1) What to edit when you are ready
--------------------------------------------------
Open: frontend/js/adsense.js

Replace these placeholders:
- clientId: ca-pub-XXXXXXXXXXXXXXXX
- homeSlot: 0000000000
- postSlot: 0000000001
- genericSlot: 0000000002

With real values from your AdSense account:
- Publisher ID (client): ca-pub-XXXXXXXXXXXXXXX
- Ad slot IDs for each unit

Tip:
- Use one slot for home, one for post page, and one shared slot for all other pages.

--------------------------------------------------
2) Google account / AdSense setup
--------------------------------------------------
1. Go to https://www.google.com/adsense/start/
2. Sign in with your Google account
3. Add your website domain
4. Complete identity and payment profile
5. Complete site ownership verification if requested
6. Wait for AdSense site approval

--------------------------------------------------
3) Create ad units in AdSense
--------------------------------------------------
After approval:
1. AdSense -> Ads -> By ad unit
2. Create Display ad units
3. Copy each generated ad slot ID
4. Paste those IDs into frontend/js/adsense.js

--------------------------------------------------
4) Deploy + verify
--------------------------------------------------
1. Deploy website to your real domain (not localhost)
2. Open pages and check browser console for errors
3. Check AdSense dashboard for site/ad-unit status

Note:
- Ad serving can take time after setup/approval.
- If you target EEA/UK users, use a consent banner/CMP compatible with Google policies.

--------------------------------------------------
5) Current implementation summary
--------------------------------------------------
- Home page uses: homeSlot
- Post page uses: postSlot
- All other pages use: genericSlot (including admin analytics pages and author page)
- Shared script: frontend/js/adsense.js

--------------------------------------------------
6) Latest platform updates (ad compatibility note)
--------------------------------------------------
- New home widgets (featured rotator + release/event strip) are ad-compatible and do not require extra AdSense config.
- New analytics detail pages (posts/categories/authors rankings) are covered by genericSlot behavior.
- Broken-link Checker page (/admin/analytics-links.html) and Newsletter admin page (/admin/newsletter.html) are also covered by genericSlot behavior.
- Footer newsletter form UX changes (desktop alignment + mobile placement) do not require any AdSense code changes.
- Profile page links layout update (3 links top row + 3 bottom row on desktop) does not require any AdSense code changes.
- In localhost/dev, AdSense initialization is intentionally skipped to reduce non-actionable CORS/cookie console noise.
- Post page YouTube embeds use privacy-enhanced `youtube-nocookie` mode; this is independent from AdSense slots.
- Posts without image now use a default fallback image (frontend/assets/default-post.svg); this does not affect ad rendering.
- Custom 404 page exists for non-API routes; keep AdSense placements on normal content pages only.
